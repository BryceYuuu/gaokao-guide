# 位次模型与轻量模拟

当用户询问冲稳保、录取可能性、滑档风险、志愿顺序压力测试，或提供带近年录取位次区间的候选 CSV 时，读取本文件。

## 模型边界

本 Skill 默认不内置完整官方录取数据库，也不声称拥有已回测校准的录取概率模型。轻量位次模拟只能帮助检查梯度、保底厚度和志愿顺序压力，不能当作录取承诺。

建议原话说明：

> 这是基于候选表中“近年位次区间”的轻量模拟，用于检查梯度和滑档风险。它没有接入完整官方招生计划和回测校准模型，不能理解为精确录取概率。

## 推荐 CSV 字段

- 学校
- 专业组或专业
- 冲稳保
- 近年位次，例如 `30000-36000`
- 计划数
- 证据状态
- 风险
- 待核验

## 模拟逻辑

对每个候选：

1. 解析近年位次区间，例如 `30000-36000`。
2. 使用区间中点作为预期门槛。
3. 使用半区间作为波动强度，并设置最低波动，避免过度确信。
4. 随机模拟年度门槛波动。
5. 如果考生位次优于或等于模拟门槛，则该候选在本轮模拟中可录取；位次数字越小越好。
6. 按志愿顺序依次判断，记录首个可录取候选。

以下情况要降低置信度：

- 证据状态为第三方、缺失或冲突；
- 计划数小于等于 3；
- 专业组下限未知；
- 当年官方招生计划缺失；
- 专业组结构与往年不同。

## 建议分层

| 模拟可达性 | 标签 | 含义 |
| --- | --- | --- |
| 大于等于 0.75 | 保/强稳 | 只有在专业组下限也可接受时，才能承担保底职责 |
| 0.45-0.75 | 稳 | 主力区间，仍需核实计划变化和专业组结构 |
| 0.10-0.45 | 冲 | 机会项，不能承担安全职责 |
| 小于 0.10 | 观察/剔除 | 除非有特殊理由，否则不占关键志愿位 |

## 脚本用法

```bash
python3 gaokao-guide/scripts/rank_simulator.py examples/sample_candidates.csv --student-rank 34500 --runs 5000 --output examples/sample_candidates.simulation.json
```

读取输出时重点看：

- 每个候选的模拟分层；
- 按志愿顺序计算的首个录取分布；
- 这个候选池里的无录取压力；
- 信源、计划数、专业组下限等风险提示；
- 模型限制说明。

## 输出纪律

- 不把模拟值包装成最终录取概率。
- 必须展示模型限制。
- 如果保底层不足或专业组下限未知，要明确说“这张表还不能作为最终志愿表”。
