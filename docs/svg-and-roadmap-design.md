# 样例 SVG 与数据接入路线图设计稿

这份文档给设计图使用，不是填报建议。建议做成两张图：一张“高考志愿方案看板 SVG 样例”，一张“数据接入路线图”。

## 图 1：高考志愿方案看板 SVG 样例

### 标题

高考志愿方案看板

### 副标题

江苏 2026 物理类 | 位次约 34500 | 模拟稿

### 视觉风格

- 画布：竖版 1200 x 1600
- 背景：浅灰蓝 `#F6F8FB`
- 主色：深蓝黑 `#172033`
- 强调蓝：`#2563EB`
- 绿灯：`#16A34A`
- 黄灯：`#D97706`
- 红灯：`#DC2626`
- 卡片：白底，轻描边，圆角 16-20px
- 字体：苹方 / 微软雅黑 / 系统无衬线
- 信息密度：一屏看完，不放长段落

### 信息层级

1. 顶部深色标题区：标题 + 副标题
2. 考生基础信息条：省份、年份、科类、分数/位次、预算、策略
3. 总览状态卡：录取安全、专业匹配、城市偏好、最大风险
4. 左侧候选池分层：冲、稳、保、观察、剔除
5. 右侧专业路径：计算机/AI、电子信息、电气、自动化
6. 风险热力图：专业组下限、招生计划变化、热门专业溢价、高价项目
7. 底部待核验清单：一分一段、招生计划、专业组全部专业、招生章程
8. 底部免责声明：仅作决策支持，不构成录取承诺

### 文案

| 模块 | 状态 | 一句话判断 |
| --- | --- | --- |
| 录取安全 | 黄灯 | 有方案，但保底看专业组下限 |
| 专业匹配 | 绿灯 | 电子信息/电气/自动化适配 |
| 城市偏好 | 黄灯 | 热门城市会压缩选择空间 |
| 最大风险 | 红灯 | 服从调剂进排斥专业 |

候选池分层：

| 层级 | 候选方向 | 风险 |
| --- | --- | --- |
| 冲 | 热门城市强专业 | 黄/红 |
| 稳 | 公办工科主线 | 黄/绿 |
| 保 | 真可接受保底 | 绿 |
| 观察 | 低估机会 | 灰/黄 |
| 剔除 | 不可接受组 | 红 |

专业路径：

| 方向 | 匹配 | 建议 |
| --- | --- | --- |
| 计算机/AI | 中-高 | 可冲，不做唯一主线 |
| 电子信息 | 高 | 主线 |
| 电气 | 高 | 主线 |
| 自动化 | 中-高 | 主线备选 |

风险热力图：

| 风险 | 等级 | 处理 |
| --- | --- | --- |
| 专业组下限 | 高 | 逐组核验，不合格不服从 |
| 招生计划变化 | 中 | 查当年计划和计划变更 |
| 热门专业溢价 | 中 | 给同层次替代 |
| 高价项目 | 高 | 超预算直接剔除 |

### 可直接生成的 SVG 数据源

使用 `examples/svg-summary-input.json`：

```bash
python3 gaokao-guide/scripts/render_summary_svg.py examples/svg-summary-input.json --output examples/gaokao-summary.svg
```

## 图 2：数据接入路线图

### 标题

gaokao-guide 数据接入路线图

### 副标题

从手工问诊到官方数据、志愿卡导入、候选池审计和信息图交付

### 设计结构

建议做成横向 5 阶段流程图，每一阶段一张卡片。卡片之间用箭头连接。每张卡片包含“输入、处理、输出”三行。

### 阶段 1：问诊与用户输入

输入：

- 中等长度问诊表
- 分数/位次/选科/批次
- 城市、专业、预算、排斥项
- 志愿卡截图或 Excel

处理：

- 识别硬约束
- 识别偏好和风险偏好
- 判断是否可进入粗筛

输出：

- 考生画像
- 缺失字段清单
- 初始风险标签

### 阶段 2：官方数据接入

输入：

- 省级考试院一分一段
- 当年招生计划
- 院校专业组
- 招生章程
- 近年投档线/录取位次

处理：

- 标准化省份、年份、科类、批次
- 标记来源等级
- 对齐分数、位次、计划数、专业组

输出：

- 官方证据账本
- 待核验清单
- 可计算候选池

### 阶段 3：第三方候选表导入

输入：

- 志愿卡导出 CSV
- Excel 候选表
- 家长截图整理

处理：

- 字段识别
- 重复项检查
- 弱信源检查
- 保底层厚度检查
- 风险字段缺失检查

输出：

- `sample_candidates.audit.json`
- 剔除/降级建议
- 需要回官方核验的字段

### 阶段 4：策略引擎

输入：

- 官方证据账本
- 候选池
- 家庭约束
- 学生偏好和执行风险

处理：

- 冲稳保分层
- 低估机会扫描
- 过热风险审计
- 专业组下限审计
- 主方案/对冲方案/机会方案

输出：

- 看板摘要
- 风险热力图
- 候选池矩阵
- 志愿排序草案

### 阶段 5：交付与复核

输入：

- 看板数据
- 最终候选表
- 待核验项

处理：

- 生成一页式 SVG 信息图
- 输出最终复核清单
- 标注信息有效期和免责声明

输出：

- `gaokao-summary.svg`
- 最终志愿表
- 风险总账
- 复核节点

### 路线图底部说明

短期：先支持手工问诊、CSV 审计、SVG 信息图。

中期：接入省级考试院公开表、招生计划和高校招生章程。

长期：建立跨省年份标准化数据库、历史回测、概率校准和专业组级模拟。
